## types/ ‚Äî Type definitions (layer 1)

### types/index.ts
**Role:** Barrel export ‚Äî re-exports all public types from sub-modules.
**Flags:** None.

### types/core.ts
**Role:** Foundational types ‚Äî `ModuleId`, `ModuleStatus`, `ModuleProgress`, `VideoProject`, `PressHandler`, `ChangeHandler`.
**Flags:**
- `ModuleId` union includes `'voice-cloner'`, `'script-generator'`, `'video-renderer'` ‚Äî future modules not yet implemented. Intentional forward-planning but technically anticipatory dead code.

### types/kanban.ts
**Role:** All Kanban types ‚Äî `KanbanItem`, `KanbanStatus`, `KanbanAction`, `KanbanContextValue`, component props, `RawPipelineRow`.
**Flags:**
- `KanbanAction` union includes `MARK_IN_REVIEW`, `MARK_DONE`, `FLAG_OUTDATED`, `CLEAR_OUTDATED` ‚Äî these action types are defined but never dispatched through the reducer; mutations go directly to DB via callbacks. The `dispatch` in `useKanban.tsx` is a no-op (`() => {}`). **Dead code** in the action type union.

### types/arc-assembler.ts
**Role:** Types for Arc Assembler ‚Äî `ArcAssemblerMode`, `ArcAssemblerOutput`, `ArcAssemblerContextValue`.
**Flags:** None.

### types/export.ts
**Role:** `PipelineExport` interface for JSON serialisation.
**Flags:** None.

### types/navigation.ts
**Role:** Route paths and navigation parameter types.
**Flags:**
- `AppRoute` type includes routes that no longer exist in the app: `'/style-selector/[order]'`, `'/style-selector/results'`, `'/scene-segmentation/input'`, `'/scene-segmentation/scene-mapper'`, `'/scene-segmentation/subject-mapper'`. These reference the old questionnaire-based style flow and the old scene-mapper route structure. **Stale dead route references.**

### types/scene-mapper-gestures.ts
**Role:** All gesture types for the frozen scene-mapper system.
**Flags:** None (frozen per architecture).

### types/scene-segmentation.ts
**Role:** Types for scene segmentation ‚Äî `Scene`, `Word`, `Subject`, `SubjectCategory`, `SegmentationConfig`, etc.
**Flags:** None.

### types/style-matcher.ts
**Role:** Types for the **old** questionnaire-based style flow ‚Äî `StyleQuestion`, `StyleOption`, `UserChoice`, `StylePrompt`, `StyleImage`.
**Flags:**
- These types belong to the old question-flow system which has been replaced by the gallery-based Style Selector. The only current consumer is `lib/styleMatcher.ts` (also dead ‚Äî see below). **Dead type definitions** unless the questionnaire flow is revived.

### types/style-selector.ts
**Role:** Types for the current gallery-based Style Selector ‚Äî `StyleSelection`, `FilterState`, `StyleSelectorContextValue`.
**Flags:** None.

### types/ui.ts
**Role:** Shared UI component types ‚Äî `ButtonProps`, `TagProps`, `ModuleCardProps`.
**Flags:** `ModuleCardProps` references `ModuleStatus` from `core.ts` ‚Äî but the actual `ModuleStatus` type (`'not-started' | 'in-progress' | 'completed'`) does NOT match the Kanban status system (`todo | up-next | in-progress | in-review | done`). This type is for a different component (`ModuleCard`, not `UniversalModuleCard`) and may refer to a deleted/never-built component.
