## Task 1 ‚Äî Card Dependency & Unlock Chain (1D)

### What needs to happen

The 4 stage cards (Style Selector, Beat Butcher, Entity Editor, Arc Assembler) must unlock in sequence. Currently all cards are static ‚Äî status never changes. This task makes status dynamic and reactive.

### Unlock rules

```
On project creation:
  Style Selector ‚Üí UP_NEXT
  Beat Butcher ‚Üí UP_NEXT
  Entity Editor ‚Üí TODO (locked)
  Arc Assembler ‚Üí TODO (locked)

When Beat Butcher reaches IN_REVIEW:
  Entity Editor ‚Üí UP_NEXT

When Entity Editor reaches IN_REVIEW:
  Arc Assembler ‚Üí UP_NEXT

Style Selector unlocks independently ‚Äî it has no downstream dependency trigger.
```

### `markInReview` ‚Äî wire to Continue button

Each card's work screen has a Continue button via `ScreenLayout`'s `onContinue` prop. Currently it just calls `router.back()`. Wire it to call `stageCallbacks.markInReview(moduleId)` which already exists in `lib/stageCallbacks.ts`.

Screens to update:
- `app/style-matcher/index.tsx` ‚Äî `onContinue` ‚Üí `markInReview('style-selector')`
- `app/scene-segmentation/beat-butcher.tsx` ‚Äî `onContinue` ‚Üí `markInReview('beat-butcher')`
- `app/scene-segmentation/entity-editor.tsx` ‚Äî `onContinue` ‚Üí `markInReview('entity-editor')`
- `app/arc-assembler/index.tsx` ‚Äî `onContinue` ‚Üí `markInReview('arc-assembler')`

Check how `stageCallbacks` is currently used in these screens ‚Äî some may already import it, some may not.

### `markInReview` logic in `useKanban`

When `markInReview(moduleId)` is called:
1. Set that card's status to `IN_REVIEW` in `card_statuses`
2. Check the unlock chain ‚Äî if the next card in sequence is `TODO`, move it to `UP_NEXT`
3. Persist via `updateProject(projectId, { card_statuses: JSON.stringify(next) })`

The module order is: `['style-selector', 'beat-butcher', 'entity-editor', 'arc-assembler']`

Beat Butcher's downstream is Entity Editor. Entity Editor's downstream is Arc Assembler. Style Selector has no downstream dependency.

### `markDone` ‚Äî lock a card permanently

Add a `markDone(moduleId)` action to `useKanban`. It sets the card to `DONE` status. A "Mark as Done" button should appear on the card only when its status is `IN_REVIEW`. This button lives on the `UniversalModuleCard` ‚Äî add it as a small secondary button below the main card content, visible only when `status === 'IN_REVIEW'`.

### Outdated badge

When a card in `IN_REVIEW` or `DONE` is re-entered and its upstream data changes, downstream cards should show an outdated indicator.

Implementation:
- Add an `isOutdated: boolean` flag to the `card_statuses` JSON per card
- When `markInReview` is called on an upstream card that was previously `IN_REVIEW` (i.e. the user went back and re-did it), set `isOutdated: true` on all downstream cards that are `IN_PROGRESS` or `IN_REVIEW`
- The outdated flag clears when that downstream card calls `markInReview` again
- On `UniversalModuleCard`, render a small "OUTDATED" pill badge (use `colors.accentAlt` ‚Äî sunglow yellow) when `isOutdated` is true

### Card status progression rules

```
TODO ‚Üí UP_NEXT (via unlock chain only, never manually)
UP_NEXT ‚Üí IN_PROGRESS (automatically when the user opens the card, i.e. on screen mount)
IN_PROGRESS ‚Üí IN_REVIEW (via Continue button / markInReview)
IN_REVIEW ‚Üí DONE (via "Mark as Done" button / markDone)
IN_REVIEW ‚Üí IN_PROGRESS (if user re-enters and makes changes ‚Äî detect via any updateProject call on that card's data column)
```

For the `UP_NEXT ‚Üí IN_PROGRESS` transition: add a `markInProgress(moduleId)` action. Call it in each work screen's `useEffect` on mount (only if current status is `UP_NEXT`).

## Task 2 ‚Äî Delete Mechanic on Project Cards

### What it does

Long-pressing a project card (not a stage card) reveals a single-row action strip below the card with one option: **Delete**.

### Behavior
- Long-press (500ms) on a project card ‚Üí action strip slides in below the card with a "DELETE" button
- Tapping outside or swiping away dismisses the strip without action
- Tapping DELETE ‚Üí shows a confirmation Alert: "Delete this project? This cannot be undone." ‚Üí confirm ‚Üí calls `deleteProject(projectId)` in `lib/database.ts`
- `deleteProject` does `powerSyncDb.execute('DELETE FROM pipelines WHERE id = ?', [id])`
- After deletion the card disappears from the board (watch() handles this reactively)
- Only one card can have the action strip open at a time ‚Äî opening another closes the previous

### Design constraints
- Action strip uses `colors.error` (alizarin crimson `#d72a21`) background
- DELETE label in `colors.text.inverse` (white), `typography.button`, uppercase
- Strip height: 44px
- Strip has 3px border in `colors.border`, `borderRadius.md`
- Hard shadow (`shadows.hard`)
- Animation: slide down from behind the card using Reanimated `useSharedValue` + spring
- This is on project cards only ‚Äî `moduleId === 'project'` ‚Äî not stage cards

### Where to add it
- `components/kanban/KanbanCard.tsx` or `components/ui/card/UniversalModuleCard.tsx` ‚Äî whichever renders the tappable card. Check which one is used for project cards specifically.
- Add a `onLongPress` handler and local `isActionOpen` state
- The action strip renders absolutely below the card

## Task 3 ‚Äî Kanban Card Design Refinement

### Current problem
Cards are white with a filled page-box containing text. They feel heavy and generic. The goal is to make them feel more neobrutalist and premium without restructuring them.

### What to tweak (designer eye, not rewrite)
- **Background:** Keep `colors.surface` (white) but ensure the 3px `colors.border` border is prominent and the hard offset shadow (`shadows.medium`) is visible
- **The filled page box:** If there is a filled background container inside the card wrapping the description text, remove the fill ‚Äî make it transparent or use a subtle `colors.background` tint instead
- **Typography:** Title should be `typography.subtitle` bold, description should be `typography.caption` in `colors.text.secondary`
- **Status pill:** Make sure the status badge is visually tight ‚Äî small pill, uppercase, correct status color, not oversized
- **Progress bar:** If present, keep it but make it thinner (3-4px height max)
- **Spacing:** Reduce internal padding if cards feel bloated ‚Äî use `spacing.sm` or `spacing.md` not `spacing.lg` inside cards
- **Icon:** The Feather icon on each card should be `colors.text.secondary`, size 18, not oversized

Do not change the card's structural layout. Tweak values only.

## Task 4 ‚Äî Kanban Column Swiping Feel

### Current problem
Swiping between columns feels long and sluggish. The next column isn't visible at all so there's no spatial sense of where you're going.

### Fixes

**Peek next/previous column:**
- The active column should take ~85% of screen width
- The next column should peek at ~15% on the right edge
- This gives spatial context and makes swiping feel purposeful
- Implement by adjusting the column width in `KanbanBoard.tsx` or `KanbanColumn.tsx`

**Cards less wide:**
- Inside each column, cards should have `marginHorizontal: spacing.sm` so there's breathing room and they don't feel wall-to-wall

**Snap behavior:**
- Ensure the horizontal ScrollView or FlatList has `snapToInterval` or `pagingEnabled` set correctly so columns snap cleanly
- If using `ScrollView`, add `decelerationRate="fast"` and `snapToAlignment="start"`

Do not change the column data logic ‚Äî visual/layout only.

## Task 5 ‚Äî Style Selector: Natural Aspect Ratio + View Toggle

### Current problem
Collage images are cropped to a square. The natural aspect ratio of the collages should be preserved. Additionally there's no way to control how many columns are shown.

### Fix 1 ‚Äî Natural aspect ratio
In `components/style-selector/CollageImage.tsx`:
- Remove any fixed height or `aspectRatio: 1` constraint
- Use `resizeMode="contain"` or let the image define its own height
- The image width fills the column width. Height is derived from the image's natural aspect ratio.
- Use React Native's `Image.getSize()` or set a known aspect ratio from the collage data if available
- If aspect ratio is unknown, use a default of `16/9` as fallback ‚Äî most collages are landscape

### Fix 2 ‚Äî View toggle (1, 2, or 3 columns)
Add a view toggle to the Style Selector screen (`app/style-matcher/index.tsx`):
- Three icon buttons in the header area: single column, two columns, three columns
- Use Feather icons: `"square"` for 1col, `"grid"` for 2col, a custom layout for 3col (or just use text "3")
- Default: 2 columns
- State: `numColumns: 1 | 2 | 3` in local useState
- Pass `numColumns` to the FlatList `key={numColumns}` (required when changing numColumns on FlatList) and `numColumns={numColumns}`
- The toggle buttons use `colors.primary` when active, `colors.text.secondary` when inactive
- Toggle sits in the top area of the screen above the gallery, below the breadcrumb header
- Style the toggle row: `flexDirection: 'row'`, `gap: spacing.sm`, aligned right
